# Creating custom ESLint rules for your design system

The accompanying codebase for the "Creating custom ESLint rules for your design system"

Which can be found here (add link).

The `main` branch acts as the starter branch, which you can follow along too when reading the article.

The `1-first-rule-complete` branch is the branch that contains the complete code for the `no-inline-style` rule.

## Getting Started

Clone the repo:

`git clone git@github.com:andrico1234/custom-eslint-tutorial.git`

Navigate inside the directory:

`cd custom-eslint tutorial`

Install dependencies:

`npm i`

Start the test suite

`npm run test`
